5、知识总结

5.1 核心命令汇总

命令 作用 使用场景
git init 初始化Git仓库 开始版本管理
git add . 添加所有文件到暂存区 准备提交代码
git commit -m “message” 提交代码并添加注释 保存版本记录
git checkout -b 分支名 创建并切换分支 开发新功能
git merge 分支名 合并分支到当前分支 功能开发完成后合并
ssh porh@IP SSH远程登录 连接虚拟机
scp -r ./* porh@IP:/path/ 上传文件到远程服务器 部署网站文件
sudo systemctl status nginx 查看Nginx状态 检查服务是否运行
sudo systemctl reload nginx 重载Nginx配置 配置文件更改后生效
sudo nginx -t 测试Nginx配置语法 检查配置是否正确

5.2 Nginx站点配置要点

配置项 作用 示例值
listen 监听的端口 80
server_name 服务器域名或IP _（匹配所有）
root 网站文件根目录 /var/www/mywebsite
index 默认首页文件名 index.html

5.3 文件与目录权限说明

权限值 含义 适用场景
755 所有者可读写执行，组用户和其他用户可读执行 网站目录
644 所有者可读写，组用户和其他用户只读 网站文件
命令 作用
chown user:group 修改文件所有者和组
chmod 修改文件权限

6、出现问题

问题 原因 解决方案
ssh: connect to host 192.168.57.100 port 22: Connection timed out IP地址输入错误，实际IP为192.168.57.100 使用正确的IP地址
scp: dest open “/var/www/mywebsite/index.html”: Permission denied 目录所有者不是当前用户，porh用户无写入权限 执行sudo chown -R porh:www-data /var/www/mywebsite
浏览器访问显示的是另一个网站（旧站点） sites-enabled目录中default和my-site同时启用，覆盖了mywebsite配置 执行sudo rm /etc/nginx/sites-enabled/default和sudo rm /etc/nginx/sites-enabled/my-site，仅保留mywebsite
修改authorized_keys时报Permission denied 对/home/webdeploy/.ssh/目录没有写入权限 改用有sudo权限的porh用户操作
sudo命令提示权限不足 webdeploy用户不在sudoers列表中 改用有sudo权限的porh用户操作

7、心得体会

通过本次综合实验，我完整经历了从开发到部署的软件工程全流程。在技术层面，我掌握了使用VS Code进行Web开发、利用Git进行版本管理和分支协作、在Linux系统中安装配置Nginx服务器，以及通过SSH和SCP实现远程管理与文件传输。这些工具的组合使用让我对现代Web开发工作流有了清晰的认识，尤其是Git分支开发模型（创建功能分支→开发新特性→合并回主分支），让我体验到团队协作开发的标准流程。

在思政层面，本次实验使用的所有工具——VS Code、Git、Ubuntu、Nginx、OpenSSH——均为开源软件。它们不仅功能强大、完全免费，而且拥有活跃的全球社区支持。这让我深刻体会到开源文化的共享精神：世界各地的开发者共同维护、持续改进这些工具，推动技术进步。我也了解到，虽然国内有Gitee、Coding等代码托管平台，并在逐步发展，但在生态建设、国际化程度上与GitHub仍有差距。作为未来的软件工程师，我应当积极学习和使用开源工具，参与开源社区，为国产软件生态的发展贡献力量。
