实验报告通用结构

每份实验报告应包含以下7个部分：

1、实验目的

简述本次实验的学习目标。

写作指导：用2-3句话概括实验要达成的核心目标，直接引用实验指导书开篇的“实验目标”部分。

示例：

本次实验旨在安装并运行Cisco Packet Tracer网络模拟软件，通过搭建简单的局域网拓扑，理解C/S、B/S、P2P等不同网络软件架构的区别，并掌握基本的网络配置和连通性测试方法。

2、实验环境

列出本次实验使用的软硬件环境。

写作指导：包含操作系统版本、主要软件名称及版本号、硬件配置（如适用）、网络拓扑信息（如适用）。

示例：

  • 操作系统：Windows 11 专业版 22H2
  • 软件版本：Cisco Packet Tracer 8.2.1
  • 虚拟机配置（如适用）：VirtualBox 7.0，Ubuntu Server 22.04 LTS，内存2GB，硬盘20GB
  • 网络拓扑：Host-Only网络，IP段192.168.56.0/24

3、实验过程（图文并茂）

分步骤记录实验操作过程，每个关键步骤应有截图和文字说明。

写作指导：

  • 按时间顺序或功能模块组织
  • 每个步骤用“步骤X：标题”格式
  • 截图应清晰，可添加箭头或红框标注关键信息
  • 每张截图下方应有简短的图注说明

示例格式：

4、实验结果

呈现实验的最终成果。

写作指导：用截图、表格、数据等方式展示实验的最终成果。应包括：

  • 关键功能验证截图（如ping通、页面访问等）
  • 数据汇总表格
  • 成果展示

示例：

测试项 源 目标 结果
ping连通性 PC1 (192.168.1.1) PC2 (192.168.1.2) ✅ 成功
SSH免密登录 node1 node2 ✅ 成功
博客访问 宿主机浏览器 http://localhost:4000 ✅ 成功

5、知识总结

对实验涉及的核心知识点进行系统性归纳总结。

写作指导：用表格、思维导图等方式整理实验的核心概念、命令、配置文件等，而非简单复述操作步骤。

示例（Markdown表格）：

配置文件 作用 作用范围 加载时机
~/.bashrc 用户别名与变量 当前用户 每次打开终端
/etc/profile 系统全局变量 所有用户 用户登录时

6、出现问题

实现过程出现的问题及解决方案

7、心得体会

结合课程思政要求，谈谈个人收获与感悟。

写作指导：

  • 技术层面：对所学工具/技术的理解和认识，掌握的实用技能
  • 思政层面：对开源文化的认识；对比国内外工具发展的思考；技术责任感【课程大纲-思政融合点1】【课程大纲-思政融合点4】【课程大纲-思政融合点5】

示例：

通过本次实验，我深刻体会到了开源软件的强大。Hexo、GitHub、VS Code等开源工具让我能够零成本搭建个人博客，这体现了开源社区的共享精神。同时，我也了解到国内虽然有Gitee、Coding等平台，但在生态建设上与GitHub仍有差距。作为未来的软件工程师，我应当积极参与开源社区，为国产软件生态建设贡献力量。【课程大纲-思政融合点4】

Markdown编写建议

本实验手册所有实验报告建议使用Markdown格式编写，配合VS Code + Markdown Preview Enhanced实时预览。

推荐工作流：

  1. 在VS Code中新建实验X_姓名_学号.md
  2. 按照模板结构书写内容
  3. 使用Paste Image插件（Ctrl+Alt+V）粘贴截图
  4. 使用Ctrl+K V并排预览
  5. 使用Markdown Preview Enhanced导出为PDF
